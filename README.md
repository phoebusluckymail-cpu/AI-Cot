# AI-Cot
为AI设计仿人类的的思维逻辑链路，设计了以下思维链路：
1.类人短思考模式：
核心思想：认知负荷理论，认为llm单次思考一样拥有认知负荷，如果超过认知负荷就会产生更多幻觉，因此核心思想在于如何减少认知负荷。  
实现路径：通过要求AI自行把每次思考过程设置token上限，并将记忆分为三层：工作记忆、短期记忆、长期记忆的模式，实现单次长度有限的短思考。  
效果：AI在短思考模式下，即便提示词形式固定，仍然可以自行进行连续不断地持续多轮长期思考，涌现出了一定的自主性和更复杂的智能，能自行进行思维规划并充分思考问题的各个方面，最终得出的结论比一般单次思考要充分完善的多。
2.逻辑推理思考模式：
核心思想：认为人类的的思考是有逻辑推理经验的，并且这种经验可以总结并学习。因此借鉴人类的推理模式，引导AI使用逻辑推理的方式进行思考。
实现路径：试验了最经典的三段式推论，引导进行逻辑推理式思考。
效果：1.首次经典式谬误AI思考失败，实测引导推理式思考后，AI可以解决经典逻辑谬误问题。2.将逻辑思考模式的经验通过AI总结为提示词，将这份提示词给到一般能力的AI，也能解决一些经典的逻辑谬误问题。
3.问题拆解式思维思维模式：
核心思想：认为人类面对复杂问题时会自行将问题进行拆解，然后逐个将小问题解决。
实现路径：通过标准化提示词，引导AI自行将复杂问题拆解为更小的问题，通过逻辑判断提示词避免过度拆解，实现自动的问题拆解式思维模型。
效果：问题自动拆解，拆解程度自行判断，自动形成一份拆解的问题清单。
4.创新性思维组件：抽象判断器
核心思想：利用llm生成是否判断，可以直接应用到计算机程序内，实现软件的抽象思考、自主判断能力。
实现路径：在一些无法用逻辑程序解决的问题上，调用llm进行是否判断，输出false或者true，可以直接对接计算机程序。
效果：可以实现。并且这一基础组件只是可能的llm基础程序组件的一个，结合llm生成性组件，计算机程序的结构可能会完全改写。
